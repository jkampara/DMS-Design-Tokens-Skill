DMS Design Tokens Agent Skill

The foundation

Built on semantic tokens

Every value the bundle paints — color, type, spacing, radii — is a semantic token, not a literal. Each one is described in references/token-roles.md — the agent reads what a token means and when to reach for it right alongside the recipes. Flip data-theme at any level and the same recipes repaint themselves; swap the token JSON and the whole bundle re-skins — the recipes don’t change.

01

Semantic, not literal

Tokens are named by role — --color-surface-1, --color-on-surface-1, --color-fill-accent-1 — never by raw value. Recipes never reach for #066AFE directly.

02

Scope-anywhere theming

Apply data-theme="day" or "night" on <html>, on a single <section>, or on a multi-blade region wrapper. The cascade does the repaint.

03

Primitives → aliases, by design

Every family ships in two layers: a primitive ladder (the brand’s tuning surface — space.5, radius.2) and aliases on top that name intent (space.layout.gutter, radius.button). Product code binds to aliases; a brand fork re-tunes the primitive ladder once and every alias inherits.

The pitch in two cards

What it is, and why we built it

01 — What

A self-contained brand bundle, hosted at a single URL

Every token (color, typography, spacing, grid, radii) and every recipe (hero, card, CTA, page-shell, gradient regions) lives in one published bundle. Agents fetch what they need on demand and follow the rules the bundle ships with. No SDK, no install, no per-project setup — just a URL and a skill entrypoint.

02 — Why

So the brand survives the AI page-building workflow

Agents that don’t know the brand invent visual values: ad-hoc hex colors, off-grid spacing, wrong type ladders. The skill replaces invention with retrieval — the agent reads the canonical recipe, copies the canonical CSS, and audits its own output against the brand contract before sending. The brand stays the brand, even when nobody on the design team sees the draft.

Who it helps

Benefits

Faster builds, higher fidelity, fewer review cycles. The skill turns “implement the brand refresh” from a multi-week alignment exercise into a fetch-and-follow workflow.

Engineers

Save hours per component

Skip the “which hex? which radius? which spacing step?” research loop. The bundle resolves every visual decision to a single var(--…) lookup — fetch three JSONs, copy the recipe skeleton, ship.

AI agents & copilots

Fewer hallucinations

Every value the agent emits is verified against a machine-readable source of truth. The gaps protocol forces agents to ask instead of guess — no more invented hex codes, made-up font stacks, or plausible-but-wrong spacing.

Designers

A launchpad for new patterns

The bundle ships brand-approved, AA-validated primitives as composable building blocks. Designers can remix them into novel card types, blade layouts, and themed event pages — build something new without starting from zero.

Accessibility reviewers

Pre-built for contrast

Every color pairing is AA-validated (≥ 4.5 : 1) at build time. Background → ink → highlight triples are curated per surface so adopters never accidentally ship a sub-AA combination.

Brand & marketing

Brand-refresh expression, captured

The refreshed identity — gradients, type pairings, Day / Night system, four theming axes — lives in code, not a static PDF. Every page built with the skill ships the new brand automatically.

Five named slots + anti-hallucination guide

Prompt framework

Every effective prompt to this skill fills five named slots. Each slot has one job; together they give the agent enough context to make sensible defaults and ask the right clarifying questions.

1

Context

Pin the bundle URL and the entrypoint file. One reusable block.

Example

“The Salesforce token bundle lives at … Read SKILL.md first; it tells you how to use the rest of the bundle.”

2

Goal

What you’re building, in one sentence of plain English.

Example

“I want a landing page with a Night-themed hero and a three-card feature grid below it.”

3

Structure

Describe the parts in the order they appear — blades, cards, CTAs.

Example

“Blade 1 = Night --2col hero with two CTAs. Blade 2 = --3up card grid with Day-island cards.”

4

Deliverables

What the agent should hand back: file format, audit walk, file list, theme summary.

Example

“Ship one HTML file. Walk the 8-question audit. Tell me which recipe files you read.”

5

Constraints

Breakpoint coverage, theme defaults, anything the agent shouldn’t do or assume.

Example

“Render at 1440 / 1024 / 768 / 414 px. Default to Desktop unless I say otherwise.”

Reduce hallucination

What to include so the agent doesn’t make things up

AI agents hallucinate when they lack specifics. Below are the six most common hallucination patterns — and the one line you can add to your prompt to prevent each one.

The agent writes a hex code not in the bundle

Fix — add to your prompt

Always include “Use only tokenized values. Never invent hex codes, pixel values, or font names.” in your CONSTRAINTS slot.

Cards on a Night blade inherit dark inks (invisible CTAs)

Fix — add to your prompt

Name the card island explicitly: “Day-island cards on the Night hero”.

Agent guesses shadow / elevation values

Fix — add to your prompt

Add “Ask me about any un-tokenized concern before generating”.

Spacing looks “off” (not aligned to the 4 px grid)

Fix — add to your prompt

Name the spacing aliases: “Use space.layout.margin for outer, space.content.lg for inter-element gaps.”

Rule of thumb: if you can name the token alias, the recipe variant, and the breakpoints in your prompt, the agent has almost nothing left to guess. Specificity is the antidote to hallucination.
