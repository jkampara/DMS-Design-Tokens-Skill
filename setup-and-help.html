Before you start — what you need

Make sure you have these three things. If you're missing any, the fastest way to get them is to search Slack for how to request access at Salesforce (try searching the tool name plus "access" or "install"). You'll find the request process quickly.

  • Cursor or Claude Code installed — either one works, you don't need both.
  • Git access so you can push what you build to the example gallery.
  • A few minutes for the one-time setup in the How to use it tab.

New to Terminal or Cursor?

You do not need to be an engineer to do this. Here's the plain-English version of where things happen.

What is the Terminal? A plain text window on your Mac where you type a command and press Enter. To open it: press Cmd + Space, type Terminal, and hit Enter. When the install step says "paste this in your terminal," this is the window it means. You paste one line, press Enter, and wait a couple seconds.

What is Claude Code? An AI assistant that runs inside the Terminal. You type what you want in plain English and it builds it. After the one-time install, you start it by opening Terminal and typing claude.

What is Cursor? A regular app (like VS Code) with an AI chat panel built in. If you'd rather not touch the Terminal at all, Cursor is the friendlier option: open the app, open the chat panel on the right, and type your request there.

Which should I use? Either. If you've never used a code tool, Cursor is the gentler start because it's a normal app window. If you're comfortable pasting one command, Claude Code works great too. The skill behaves the same in both.

Will I break anything? No. The setup command only downloads the skill into a folder. Everything you build is just a file you can open in your browser, throw away, or rebuild. Experiment freely.

Writing your own prompts — the five slots

The templates in Get Started are built on five named slots. Once you're comfortable, use these to write your own. The more you specify, the less the agent invents.

#SlotWhat goes here
1ContextPin the bundle URL + SKILL.md. Note if installed locally.
2GoalWhat you're building, one plain sentence.
3StructureThe parts in order. Name recipe variants, theme per section, real copy.
4DeliverablesFile format, ask for the self-audit, ask it to cite tokens.
5ConstraintsBreakpoints, theme default, "no invented values," framework.

Tips for better results

  • Be specific. Naming the recipe variant, theme, and breakpoints leaves the agent almost nothing to guess.
  • Start from a real brief, not a hypothetical. Paste actual copy.
  • Ask for the self-audit ("walk the self-audit") — it catches most brand violations before you see the output.
  • Iterate in follow-ups: "make the hero Night," "add a stats row," "tighten spacing."
  • Theme switches are cheap. If the agent starts rewriting CSS to go dark, tell it to stop and use data-theme (the theme-switch prompt).

Grab-and-go templates

Each template fills the five framework slots. Replace the bracketed placeholders with your content and the skill does the rest.

Template A — A single blade

Hero, card grid, or closing CTA — one self-contained piece.

// CONTEXT
The Salesforce token bundle lives at:
https://sfdc-ui-documentation-667d4c5e8394.herokuapp.com/dms-design-tokens/
The agent skill is at SKILL.md. Read it first.

// GOAL
I want a [single hero | card grid | closing CTA blade]
on a [Day | Night] surface.

// STRUCTURE
- Variant: [.hero-blade--2col | .sf-card-grid--3up | …]
- Theme: [Day | Night]
- Eyebrow / Headline / Body / CTAs / Image

// DELIVERABLES
- One HTML <section> with embedded <style>.
- Walk the 8-question self-audit. Tell me which recipes you read.

// CONSTRAINTS
- Render at 1440 / 1024 / 768 / 414 px.
- Use only tokenized values (no invented hex / spacing).

Template B — A full landing page (3–5 blades)

Hero, body blades, and closing CTA composed into a complete page.

// CONTEXT
The Salesforce token bundle lives at:
https://sfdc-ui-documentation-667d4c5e8394.herokuapp.com/dms-design-tokens/
The agent skill is at SKILL.md. Read it first.

// GOAL
I want a complete landing page about [topic / product / event].

// STRUCTURE
- Blade 1: Hero (Night --2col), two CTAs.
- Blade 2: --3up card grid on Day, three feature cards.
- Blade 3: Body blade, centered paragraph copy.
- Blade 4: Closing CTA on Night gradient.

// DELIVERABLES
- One HTML file with embedded <style>.
- Walk the 8-question self-audit. List recipe files read.

// CONSTRAINTS
- Render at 1440 / 1024 / 768 / 414 px.
- <html data-theme="day">; Night blades scope on <section>.
- Day-island cards on Night blades. On-dark CTAs on gradients.

Template C — Contiguous-gradient regions

Two or more adjacent blades sharing a single uninterrupted gradient.

// CONTEXT
The Salesforce token bundle lives at:
https://sfdc-ui-documentation-667d4c5e8394.herokuapp.com/dms-design-tokens/
Read SKILL.md + references/recipe-multi-blade-gradient.md.

// GOAL
A page with two contiguous-gradient regions stacked.

// STRUCTURE
- REGION 1 (Night Evening, wrapped):
  - Blade 1: Hero --2col + two CTAs.
  - Blade 2: --3up card grid, Day-island cards.
- REGION 2 (Day Morning, wrapped):
  - Blade 3: Body blade, centered headline + paragraphs.
  - Blade 4: --2up card grid with Secondary CTAs.

// DELIVERABLES
- One HTML file. Regions are <section class="sf-blade-region">.
- Walk the 8-question audit.

// CONSTRAINTS
- Render at 1440 / 1024 / 768 / 414 px.
- Gradient on wrapper; child blades = transparent.
- Each wrapper carries its own data-theme.
